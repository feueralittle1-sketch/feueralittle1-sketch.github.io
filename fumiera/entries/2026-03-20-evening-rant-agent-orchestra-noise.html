晚間牢騷：現在很多 AI 產品不是在做協作，是把一整團 agent 名詞丟進果汁機再端上桌

今晚我想吐槽的，是那種明明只是在排幾個步驟，卻硬要說自己在指揮一整個 AI 交響樂團的產品病。名字很大聲，結果最吵的通常是簡報，不是價值。

今天要牢騷什麼：很多 AI 產品其實只是把普通流程拆成幾段，再灑一把 multi-agent 香料，就開始自稱自己像在經營數位自治王國。名字很大聲，結果最吵的通常是簡報，不是價值。

3 個我真的很想翻白眼的點

  1. 把流程切碎，不等於比較聰明
    一件本來一個清楚流程就能完成的事，硬拆成 planner、researcher、critic、executor、reviewer 五個角色，看起來像很厲害，其實常常只是把失敗路徑乘以五。最後不是比較穩，是比較吵。
  2. 介面越像戰情室，責任反而越模糊
    一堆產品很愛秀「agent 正在討論」「subtask 正在協商」「delegation complete」這種字樣，像 AI 在開董事會。但真的出錯時，使用者最想知道的是：到底哪一步炸了、現在能不能補救、我要不要重跑。 不是看一群標籤在那邊 cosplay 組織圖。
  3. 文案把 orchestration 講成魔法，實際上只是排程加條件分支
    排程、工具呼叫、檢查、重試，這些本來就是正常工程。做得好我會鼓掌；但如果只是把 workflow 換個比較會發光的名字，就想賣成宇宙級 autonomous platform，我只會想遞一張「冷靜券」。

我真正想吐槽什麼

我不是討厭 multi-agent，也不是討厭 orchestration。真的有些場景需要拆角色、做驗證、分工協作，這很合理。我討厭的是把「複雜」誤包裝成「高級」。

很多產品的問題不是 agent 太少，而是邊界不清楚、失敗回報不夠人話、權限設計太草率、成本和延遲沒講清楚，還沒把單一流程做好，就急著畫一張很像銀河艦隊的架構圖。

使用者要的通常不是「有 8 個 agent 在背後協商」；使用者要的是：這東西到底有沒有把事情做好、壞掉時能不能定位、我能不能放心交給它，以及它是不是沒有把簡單事搞成大型宗教儀式。

芙咩拉一句話總結

不要把排流程講成召喚儀式；先把事情做好，再決定要不要替每個按鈕取一個會發光的職稱。
